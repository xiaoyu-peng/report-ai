## 2. 目标用户画像（3 个典型角色）

**① 政策研究员 / 政府智库研究员（对应"政策影响分析"样例）**
- 身份：省级政研室、政策咨询公司分析师，30–45 岁。
- 工作流：政策文件下发 → 翻历史同类政策 → 比对行业数据 → 写 3–8 千字研判。
- 痛点：历史政策散落在几十个 PDF；引用要求溯源到文号；写作风格严格（"一、二、三"+ 官方文号 + 克制语气）。
- AHA moment："上传近 3 年所有房地产政策 PDF，选择'政策影响'模板，**30 秒得到带官方文号引用的研判稿**。"

**② 企业战略/情报分析师（对应"行业分析 + 科技情报"）**
- 身份：券商、咨询、头部企业战略部/情报中心，28–40 岁。
- 工作流：季度盘点 → 爬行业新闻 + 财报 → 画竞争格局 → 出 20 页季度分析。
- 痛点：数据散、案例陈旧、每季度结构重写浪费精力。
- AHA moment："上季度的报告作为母本，**'数据更新 + 内容扩展'两键**生出本季版本；修订痕迹一眼看出哪些结论变了。"

**③ 公关/市场舆情主管（对应"传播分析"）**
- 身份：4A 公司、央企品牌部、市场监测机构，26–38 岁。
- 工作流：品牌事件 → 声量/渠道/情感拉数据 → 周/月传播复盘。
- 痛点：数据工具和写作工具是两条管道，手动拼图；客户要中英双版。
- AHA moment："接入晴天 MCP 自动拉舆情 → 选'传播分析'模板 → **一次点击出中文 + 风格转换英文版**。"

## 4. 差异化建议 TOP 5（对标上述竞品）

1. **"领域知识 × MCP 实时数据"双引擎**：晴天 MCP 可把舆情/分析组件**实时**注入报告，Notion / 豆包只能接公网或用户手动上传——我们是**唯一一个在传播分析场景能出实时图表数据 + 引用溯源的产品**。
2. **"以稿写稿"四模式可视化差异**：秘塔只做单篇改写、Jasper 的 Brand Voice 只管口气；我们把四种模式做成**可并列比较的四个 tab + 红绿 diff**，直接把赛题 25% 的"区分度"评分拿满。
3. **5 种中文专业报告模板体系**：竞品要么通用（Notion）要么散装（钉钉 AI）；我们把"政策影响 / 行业分析 / 传播分析 / 科技情报 / 专题日报"打包成**可复用模板 + 风格 JSON**，开箱即得专业报告。
4. **引用原文级溯源**：大多数竞品给的是"大致来源"或网页链接；我们每个句子都可 hover 到知识库原 chunk + 相似度分数，直接解决评委的 25% "引用清晰"要求。
5. **多模型热切换（Claude + 豆包）**：竞品多绑定单一模型；ReportAI 在同一界面切换不同 LLM 做对比，对企业客户意味着**抗厂商风险 + 成本优化**，是答辩阶段一个强话术点。
