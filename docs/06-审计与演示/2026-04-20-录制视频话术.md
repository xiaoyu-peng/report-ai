## 二、技术栈介绍（约 2:00–2:30）

### 架构总览（1:50–2:30）

【屏幕】切到架构图（或用 IDE 项目结构 + 浏览器 DevTools 窗口演示）

【字幕】`前端 Vue 3 · 后端 Spring Boot · 存储 MySQL + ES + Redis`

---

### 后端技术栈（2:30–3:20）

【屏幕】
- IDE 打开 `report-ai/backend/` 展示 Maven 多模块
- 镜头扫 `report-hub-common` / `report-hub-system` / `report-hub-knowledge` / `report-hub-report` / `report-hub-api`

【口播】
> 后端是 **Java 17 + Spring Boot 3.2.5 + MyBatis Plus** 的 Maven 多模块工程，一共 5 个模块：
> **common** 放通用的 Result、JWT 工具、全局异常、Spring Security 配置；
> **system** 是从第一站复用的用户、角色、权限管理；
> **knowledge** 负责知识库，用 **Apache Tika 解析 PDF/DOCX**，切分后同时写入 **MySQL FULLTEXT** 和 **Elasticsearch**，支持中文 ngram 分词召回；
> **report** 负责报告生成与改写，核心是用 **SSE 流式输出** 把 Claude API 的 token 实时推给前端；
> **api** 是启动入口和 `application.yml`。
> 外部能力接了两个：**晴天自研 MCP**（信息检索 + 分析组件）和 **Claude API**（负责理解、生成、质检）。

【字幕】`Spring Boot 3.2.5 · MyBatis Plus · Elasticsearch · Tika · SSE · Claude API · MCP`

---

### 前端技术栈（3:20–3:50）

【屏幕】IDE 打开 `report-ai/frontend/` → 展示 `package.json` 和 `src/views/` 目录

【口播】
> 前端是 **Vue 3 + Vite + Element Plus + Pinia + Vue Router**，使用 TypeScript。
> 写作工作台用 **Tiptap** 富文本编辑器支持引用角标交互；
> PDF 预览用 **iframe + pdf.js**；
> Word 导出走后端 **docx4j** 组装，PDF 导出用前端 **html2pdf**；
> 流式生成全程走原生 **EventSource** 对接 SSE，渲染侧用响应式状态实时追加章节内容。

【字幕】`Vue 3 · Tiptap · Element Plus · Pinia · EventSource / SSE`

---

### 存储与部署（3:50–4:20）

【屏幕】终端跑 `docker ps` → 展示 MySQL / Redis / Elasticsearch 容器

【口播】
> 存储层用 **MySQL 8.0、Redis、Elasticsearch 8** 三件套。
> MySQL 建表时特别设置了 **`innodb-ft-min-token-size=1`** 保证中文 FULLTEXT 能召回单字；
> Elasticsearch 作为主检索引擎，配合 MySQL 做兜底；
> Redis 承担会话缓存。
> 部署上 MySQL、Redis、ES 跑在 Docker，前后端在 IDE 里跑方便断点调试。

【字幕】`MySQL 8 FULLTEXT · ES 8 · Redis · Docker Compose`

### 5. 以稿写稿 × 4 改写 + 续写（7:30–9:30，约 120 秒）⭐ 赛题 25%

【屏幕节奏】指令输入 2 秒 → 流式剪成 2 倍速 → 版本对比停 3 秒

#### 5.1 数据更新（20 秒）
【屏幕】改写 → **数据更新** → `替换为 2026 最新公开数据` → v2 三色 diff
【口播】赛题 25% 权重的核心——**以稿写稿**。第一种是数据更新，走 MCP 拉最新公开数据替换老数字，完成后这里就是三色 diff，新增绿色、删除红色、修改橙色。

#### 5.2 视角调整（20 秒）
【屏幕】改写 → **视角调整** → `面向地方政府领导的简报版，不超过 800 字` → v3
【口播】第二种是视角调整，从研判全文压到 800 字领导简报，整体结构被整体压缩。

#### 5.3 内容扩展（20 秒）
【屏幕】改写 → **内容扩展** → `补充 3 个 AI+制造落地案例章节` → v4
【口播】第三种是内容扩展，走 MCP 的案例检索，把正文扩到原来 1.8 倍。

#### 5.4 风格转换（20 秒）
【屏幕】改写 → **风格转换** → `正式研判 → 通俗科普` → v5
【口播】第四种是风格转换，从研判切到科普，语气肉眼可见变软。

#### 5.5 续写新章（30 秒）⭐
【屏幕】改写 → **续写新章节** → `在结尾补写"AI+ 制造未来 3 年趋势预判"章节` → 拉到正文末尾展示新增第四章
【口播】最后是赛题明确要求的**续写**。关键点是**原稿一个字不动**，末尾追加新章节并自动续编章节号——这也是我们专门修过 bug 的场景，早期版本会"新增一章把全文覆盖"，现在通过流式导入原报告加章节合并解决了。

【字幕】`4 种改写 + 1 种续写 = 赛题 25% 满覆盖`
