## 二、25 分钟演示流程

   - 知识沉淀 / 风格复用 / 质量可控 / 一稿多写
3. Dashboard 一句话带过

### 1. 模块 1 · 知识库管理（2 min）

4. 左导航 → **知识库管理**
5. 顶部一行介绍 → 指**分类 tab**（政策 / 宏观 / 行业 / 消费 / 历史 / 媒体资讯 + 其他）
6. 点开 **KB 1 政策法规库** → 展示 8 份 AI+ 政策文档
7. **现场上传 1 个 PDF**（从 `docs/01-样例报告/` 任选 < 1MB 的 PDF）
   - 解析 + chunk 数立即更新 → **"多格式批量导入"**
8. 搜索框输入 `AI+制造` → 返回命中分块带页段 → **"自动检索 + 溯源"**

### 2. 模块 3 · 模板中心（2 min）

9. 左导航 → **模板中心**
10. 表格展示 **5 份 T5 样例模板**（id 12-16）
11. 点 `T5样例-政策影响` → 右侧 Dialog 展示 `structure_json`：
    - `section_hierarchy` = 章节层级
    - `tone` = 研判
    - `data_density` = 高
    - `signature_phrases` = 高频金句
    - **"这就是 LLM 刚才学到的写作风格"**
12. 右上角**上传参考报告**按钮——点一下再取消，只为展示入口

### 3. 模块 3 · 生成主题 A ⭐（3 min，走已固化结果）

13. 左导航 → **写作工作台**
14. **预生成结果展示**：直接跳到**报告库 → 打开"AI+ 制造专项行动政策影响研判"**（报告 id 19）
    - 理由：演示时 LLM 慢，现场跑 2-3 min 不优雅。固化结果直接亮正文
15. 指**标题 / 主题 / 关键要点 / 模板 / 知识库 / 深度**配置（在报告详情 meta 区看得到）
16. 报告正文展示 `[1][2]...[n]` 角标密集（目标：8+ 处）

### 4. 模块 2 · 引用溯源验证（1 min）⭐

17. 鼠标 **hover** 任一 `[n]` → 弹出浮层（原文片段 + 页段）
18. 点击 `[n]` → 右侧引用溯源列表高亮对应项
19. 点"查看原文" → 新 tab 打开 KB 详情页 + 直接预览 PDF 对应页段

### 5. 模块 3.4 · 质量体检（1 min）

20. 报告详情顶部 tab → **覆盖度体检**
21. 点"开始质检"—期间讲解**三维度**：
    - 覆盖度（关键要点是否都落到正文）
    - 引用准确性（每个 `[n]` 能否从对应分块推出）
    - 事实性（具体数字 / 日期有没有分块支撑）
22. 等 60-90 秒 → 结果：总分 + 三分项分数 + 可疑事实列表

### 6. 模块 4 · 以稿写稿 × 4 改写（8 min）⭐ 赛题 25% 核心

在主题 A 报告详情页，**逐一**跑 4 种改写（每次 ~1.5 min）：

#### 6.1 数据更新（1.5 min）
- 点"改写" → 选**数据更新**
- 指令：`替换为 2026 最新公开数据`
- 等流式完成 → 切"**版本对比**"tab → 展示 v1→v2 三色 diff
- 指出 `diff-stats-bar`：新增 / 删除 / 修改行数

#### 6.2 视角调整（1.5 min）
- 改写 → **视角调整**
- 指令：`面向地方政府领导的简报版，不超过 800 字`
- 对比 v2→v3：整篇压缩到 800 字以内，语气变"要点先行"

#### 6.3 内容扩展（1.5 min）
- 改写 → **内容扩展**
- 指令：`补充 3 个 AI+制造落地案例章节`
- 对比 v3→v4：正文扩成约 1.8 倍

#### 6.4 风格转换（1.5 min）
- 改写 → **风格转换**
- 指令：`正式研判 → 通俗科普`
- 对比 v4→v5：语气从"研判"变"讲故事"

#### 6.5 续写（1.5 min）⭐ 赛题明列
- 改写 → **续写新章节**
- 指令：`在结尾补写"AI+ 制造未来 3 年趋势预判"章节`
- 展示：原稿全部保留 + 末尾新增"四、..."新章节
- 指章节自动续编号（前面是"三、..."，续写到"四、..."）

### 7. 模块 5 · 版本管理 + 导出（2 min）

23. 报告详情 → **版本历史** tab
    - 展示 v1 initial + v2-v6 五次改写版本（可看到 source_mode 标签）

### 8. 主题 B + 主题 C 快速补刀（3 min）

## 四、现场突发 Q&A 储备

**Q：看起来只是调用大模型，没什么难度？**
> RAG 检索是 MySQL FULLTEXT + NGRAM 分词支持中文召回；引用溯源具体到**文档名 + 页/段**；4 种改写模式每种走不同的 MCP 数据源（数据更新走最新舆情、扩展走案例搜索、视角调整走情感分布）；改写前快照持久化支持三色 diff。

**Q：为什么不做向量数据库？**
> MySQL FULLTEXT 已满足赛题"知识库驱动"+"溯源到页段"需求；NGRAM 分词解决了中文召回问题（之前召回接近 0 的 bug 已修）。演示数据 33 份文档、500+ 分块召回准确度实测够用，引入向量库反而增加部署复杂度。

**Q：质量检查是不是 LLM 说啥就是啥？**
> 三维度 `LLM-as-judge` 走 JSON schema 约束输出（covered / cited / factual 三份列表），提示词显式要求"只要核心事实能从分块看出即通过"。解析失败自动降级给一个兜底 `QualityReport`，不会 500。

**Q：如果 LLM 调用失败怎么办？**
> 系统级降级：① MCP 有 15s 超时预算，超时返 empty 不拖死 SSE；② 引用 parser 找不到有效 n 值时写警告日志；③ 质量检查 JSON 解析失败走 fallback 报告。演示时后端日志可看到这些 guard 的生效。
