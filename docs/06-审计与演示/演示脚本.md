# ReportAI 演示脚本

> 按此脚本跑一遍系统，即可产出赛题 D.2 / D.3 / D.4 / D.5 全部交付物。
> 预估用时：**3–4 小时**（其中 LLM 生成各 2–4 min/份）。

## 前置准备

### 启动系统

```bash
cd report-ai
export ANTHROPIC_API_KEY=...   # 或 DOUBAO_API_KEY（二选一）
export TAVILY_API_KEY=...      # 可选，Web 搜索兜底
docker compose up -d --build
docker compose logs -f backend  # 新 terminal，看日志
```

打开 [http://localhost:3001](http://localhost:3001)，用 `admin / admin123` 登录（见 init.sql 种子数据）。

### 老库升级（如果有历史数据）

本轮 schema 加了 `knowledge_chunk.page_start/page_end` + `report.include_keywords/exclude_keywords/generation_depth`。若不 reset 数据库，手动跑：

```sql
ALTER TABLE knowledge_chunk
  ADD COLUMN page_start INT DEFAULT NULL,
  ADD COLUMN page_end INT DEFAULT NULL;

ALTER TABLE report
  ADD COLUMN include_keywords VARCHAR(500) DEFAULT NULL,
  ADD COLUMN exclude_keywords VARCHAR(500) DEFAULT NULL,
  ADD COLUMN generation_depth VARCHAR(20) DEFAULT 'standard';
```

或直接 `docker compose down -v && docker compose up -d --build` 全 reset。

## D.2 — 3 份不同主题的完整报告（含引用来源标注）

### 主题选择

建议三份跨"行业分析 / 政策影响 / 传播分析"：

| 序 | 主题 | 建议模板 | 知识库分类 | 深度 |
| --- | --- | --- | --- | --- |
| 1 | 2025 年生成式 AI 在文旅产业的应用进展 | 行业分析 | 行业报告 | standard |
| 2 | 国家数据局《公共数据资源登记管理办法》影响分析 | 政策影响 | 政策法规 | deep |
| 3 | 春节档舆情：AI 视频生成工具进入大众视野 | 传播分析 | 媒体资讯 | brief |

### 跑一遍的操作步骤

对每个主题：

1. **知识库 → 新建**：选对应分类，名字与主题一致；**分别**上传 3–5 篇相关 PDF / DOCX（从 `docs/以稿写稿-报告样例/` 或自己准备）
2. **工作台**：
   - 填标题、核心主题、关键要点（建议 4–6 条）
   - "选择知识库" 选刚建的 KB
   - **展开"检索条件"** 加 2–3 个补充关键词、1–2 个排除词（演示赛题 2.3）
   - **生成深度** 选对应档位
   - **选择模板**（提前上传过的样稿模板；走 `/templates/analyze-file` 上传参考稿）
3. 点"开始生成报告"
4. 生成过程中：
   - 观察"引用溯源"标签页的 SSE chunks 事件推送（每条带页码、相关度）
   - 观察 token 流
5. 生成完成后自动切预览：
   - 正文 `[n]` 角标 hover → 弹原文浮层（**Granola 式**，评委彩蛋点）
   - 点击角标 → 跳右侧卡 + 高亮
6. **点"质量检查" tab → 运行检查** → 三维度评分展示
7. **头部"导出" → Word 或 PDF** → 存到 `docs/演示素材/D2-主题N/`

### 产出物清单

每主题：
- `docs/演示素材/D2-主题{N}/原报告.docx`（系统导出）
- `docs/演示素材/D2-主题{N}/质量检查截图.png`
- `docs/演示素材/D2-主题{N}/引用溯源截图.png`

## D.3 — 3 篇参考报告 × 4 种改写模式

基于上一步生成的 3 份报告（或另存 3 份参考稿），每份演示一种改写模式：

| 原报告 | 改写模式 | 指令示例 |
| --- | --- | --- |
| 主题 1（文旅 AI） | **DATA_UPDATE** | "把 2024 年数据全部替换为 2025 年 Q1 最新数据" |
| 主题 2（数据政策） | **ANGLE_SHIFT** | "改写成面向省级政务部门领导的 800 字决策简报" |
| 主题 3（春节舆情） | **EXPAND** | "补充一节'案例深度剖析'，加入 Sora 2 和可灵 3 代的典型案例" |

另加一份演示 **STYLE_SHIFT**（任选原稿）：
- 指令："把整篇从研判风格转成面向大众的科普口吻，降低术语密度"

### 操作步骤

1. 打开"报告库" → 选对应报告 → 点"改写"下拉 → 选模式
2. 输入指令 → 确认
3. 等 SSE 流完成
4. 切到"版本对比" tab → from=v1, to=v2 → **观察**：
   - 顶部紫色"变更背景卡"会显示：模式 + 指令摘要 + MCP 来源 + 字数变化（**P2 差异化亮点**）
   - 下方统计条：新增 N / 删除 N / 修改 N
   - 切"修订痕迹"视图看段落级红绿标记
5. 截图存 `docs/演示素材/D3-{mode}/`
6. 导出新版本 DOCX

## D.4 — 续写演示

选主题 2（政策类报告）演示续写：

1. 在报告详情，点"改写" → **"续写新章节"**
2. 输入新章节主题：例如 "四、2026 年监管展望与企业合规建议"
3. SSE 流：原稿保留不变，新章节追加在末尾（前端自动插段落分隔）
4. 观察版本对比 → change_summary 会显示："改写模式：CONTINUATION · 指令：2026 年... · MCP: 相关文章, 相关热词, Web搜索 · 4512 → 6800 字 (+2288)"
5. 截图 + 导出 DOCX 存 `docs/演示素材/D4-续写/`

## 已知风险与规避

- **MCP 远端慢/挂**：已加 20s 总预算，超时自动降级为"无 MCP 上下文"的纯 RAG 生成
- **导出中文文件名乱码**：已用 RFC 5987 `filename*=UTF-8''...`，现代浏览器（Chrome/Edge/Safari/Firefox）都 OK
- **PDF 页码不准**：Tika 的 `<div class="page">` 划分对大多数 PDF 有效；扫描件 / 图片型 PDF 可能整体归为 1 页（不做 OCR）
- **质量检查返回非 JSON**：Service 有降级逻辑，给空报告 + "模型返回非 JSON" 提示
