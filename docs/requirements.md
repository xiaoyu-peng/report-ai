# T5 智能报告写作平台 — 需求分析

> 来源：`docs/竞赛题目/AI闯关赛第二站.pdf` 第 6 页（T5 · 智能报告写作平台——知识库生成 × 以稿写稿）

## 赛题定位

业务语境：政府、企业机构的决策场景需要大量情报总结和报告产出。传统人工撰写效率低、质量不稳定、风格难统一。本赛题要求打造一个"以内部知识资产为驱动"的报告生成系统，两个核心能力：

1. **知识库驱动生成**——以用户上传的"行业知识 / 历史报告 / 内部资料"为基础，生成新选题的高质量报告；
2. **以稿写稿 / 智能改写**——上传参考报告作为"写作母本"，系统学习其结构与笔风，并在此基础上改写或扩展。

## 核心模块（5 个）

### 模块一：知识库构建与管理
- 支持批量导入文档，格式覆盖 PDF / Word / TXT / Markdown；
- 支持通过 URL 写入知识库（网页抓取）；
- 保证准确度：覆盖 政策法律 / 行业报告 / 历史数据 / 编码资料 等；
- 支持权限控制（内部编辑与管理）。

### 模块二：知识检索与引用（**核心**）
- 用户输入报告主题 / 要求时，自动从知识库检索相关内容；
- 检索结果必须 **支持溯源、原始出处、相似度**；
- 支持多 Agent 在检索条件 / 策略上互不矛盾。

### 模块三：AI 报告生成
- 基于知识库内容生成报告（知识库驱动模板）；
- 支持对比 / 参考用户上传报告进行大纲 / 风格学习，生成新主题报告；
- 大纲可单步修改；
- 报告质量保障：事实性 / 非诱导性、引用来源完整、知识库重复性分析。

### 模块四：以稿写稿 / 智能改写（**核心**）
- 上传一篇或多篇参考报告，系统自动解析原稿主体结构；
- **四种改写能力**（必须都实现且效果可区分）：

| 模式 | 说明 |
| --- | --- |
| 数据更新 | 保持结构，替换旧数据为新数据 |
| 视角调整 | 同内容，改观点 / 立场 / 受众（如行业报告 → 领导简报）|
| 内容扩展 | 基于原稿框架，AI 补充新分析、新案例、新章节 |
| 风格转换 | 正式 ↔ 通俗，中文 ↔ 英文 |

- AI 模型能力：在原稿基础上续写新章节，与原稿逻辑 / 结构 / 风格保持一致；
- **修订痕迹视图**：改写前后差异对比（新增-删除-修改一目了然）。

### 模块五：版本管理与导出
- 保存多个改写版本，支持随时回溯对比；
- 支持导出 Word / PDF。

## 交付要求

1. 可直接打开使用的完整 Web 系统（五个模块全部可用）；
2. **≥ 10 个** 不同主题的完整报告生成示例（含引用来源标注）；
3. **3 篇** 参考报告，分别演示 4 种改写模式；
4. 至少 **3 个** 改写演示（从参考稿成改写输出）；
5. 改写前后对比演示（修订痕迹视图）；
6. 设计说明文档 + AI 工具使用日志。

## 评分标准（满分 100）

| 维度 | 权重 | 评分要点 |
| --- | --- | --- |
| 知识检索准确性 | 25% | 检索结果是否准确、引用来源是否清晰 |
| 报告生成质量 | 25% | 报告结构是否专业、内容是否有深度、仿写风格还原度 |
| 改写质量 | 25% | 改写是否自留质量、四种模式是否有区分度 |
| 对比展示与版本管理 | 15% | 差异是否直观、版本管理是否易用 |
| 系统完整性与体验 | 10% | 全流程是否跑通、操作是否流畅 |

## 总体赛事要求

- 必须交付可运行的实物，不接受纯 PPT 演示；
- 必须提交 AI 工具使用日志（用了哪些工具、解决了哪些问题，无日志不得答辩）；
- 技术路线不限：Cursor / Claude Code / GitHub Copilot 任意；
- 答卷整理至一份压缩包，命名 "姓名-题目"。

## 风险与落地要点（结合样例与 MCP）

- **5 种报告类型**（详见 `report-style-guide.md`）：政策影响 / 行业分析 / 传播分析 / 科技情报 / 专题日报周报。四大核心模块均需演示跨至少 2 种类型，否则"仿写风格还原度"扣分。
- **评分 75% 集中在"检索 + 生成 + 改写"**，这三块必须做扎实；版本管理和全局体验只占 25%，不要过度打磨 UI。
- **改写 25% 权重要求四模式有区分度**，需在前端用"修订痕迹视图"把 diff 直接展示出来，而不是只给一份新文本。
- 可直接利用自研 MCP（`mcp-integration.md`）为"传播分析"和"政策影响"类报告提供舆情数据支撑，绕开自建采集管道。
