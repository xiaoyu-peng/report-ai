# T5 智能报告写作平台 — 需求分析

> 来源：`docs/竞赛题目/T5  智能报告写作平台——知识库生成、以稿写稿.docx`（官方原文）

## 背景

当前报告写作平台依赖wenhai和互联网数据生成报告，存在两个核心痛点：一是缺乏对内部知识资产的有效利用（大量历史报告、政策文档、行业资料沉睡在文件夹里）；二是"以稿写稿"完全依赖人工（晴天大模型报告参考旧报告写新版本效率极低）。本题要求在写作平台或新搭建平台的基础上，构建"知识库驱动生成"与"以稿写稿"两大核心能力，交付可直接使用的完整系统。

## 核心模块（5 个）

### 模块一：知识库构建与管理
- 支持批量导入文档(PDF/Word/TXT/Markdown)，支持通过协议外挂知识库
- 按分类管理(政策法规/行业报告/历史报告/媒体资讯)
- 支持知识库内容增删改查

### 模块二：知识检索与引用（**核心**）
- 用户输入报告主题/提纲后，自动从知识库检索最相关内容
- 检索结果标注来源文档、页码/段落，支持溯源验证
- 支持手动补充检索条件或排除不相关内容

### 模块三：AI 报告生成
- 基于知识库内容生成报告(知识库驱动模式)
- 支持仿写：参考用户上传报告的大纲与写作风格，生成新主题报告
- 大纲可保存复用
- 报告质量保障：事实性错误检测、引用准确性检查、知识库覆盖度分析

### 模块四：以稿写稿 / 智能改写（**核心**）
- 上传一篇或多篇参考报告，系统自动解析报告结构
- **四种改写模式**（必须都实现且效果可区分）：

| 模式 | 说明 |
| --- | --- |
| 数据更新 | 保持结构，替换旧数据为新数据 |
| 视角调整 | 同内容，改视角/立场/受众（如行业报告→领导简报）|
| 内容扩展 | 基于原稿框架，AI补充新分析、新案例、新章节 |
| 风格转换 | 正式↔通俗，中文↔英文 |

- AI续写能力：在原稿基础上续写新章节，与原稿逻辑、数据、风格保持一致
- 修订模式视图：改写前后差异对比（新增/删除/修改一目了然）

### 模块五：版本管理与导出
- 保存多个改写版本，支持随时回溯对比
- 支持导出 Word/PDF

## 交付需求

1. 可直接打开使用的完整Web系统（五个模块全部可用）
2. **3份**不同主题的完整报告生成演示（含引用来源标注）
3. **3篇**参考报告，分别演示4种改写模式
4. 至少**1个**续写演示（从参考稿续写新章节）
5. 改写前后对比展示（修订模式视图）
6. 设计说明文档 + AI工具使用日志

## 赛事总体评分标准（所有题目通用）

| 维度 | 权重 | 评分要点 |
| --- | --- | --- |
| 创新性 | 40% | 创新程度、技术难度、商业价值 |
| 完成度 | 25% | 功能完整性、系统稳定性 |
| Prompt 质量 | 20% | prompt 设计+优化、AI 工具使用深度 |
| 答辩表现 | 10% | 表达清晰、逻辑严密 |
| 团队协作 | 5% | 分工合理、协作高效 |

> 注：此为赛事总评标准，T5 专项评分见下方。答辩时两套标准都会参考。

## T5 专项评分标准（满分 100）

| 维度 | 权重 | 评分要点 |
| --- | --- | --- |
| 知识检索准确性 | 25% | 检索结果是否相关，引用溯源是否准确 |
| 报告生成质量 | 25% | 报告结构是否专业，内容是否有深度，仿写风格还原度 |
| 改写质量 | 25% | 改写是否保留原稿精华，四种模式是否有区分度 |
| 对比展示与版本管理 | 15% | 差异是否直观，版本管理是否完善 |
| 系统完整性与体验 | 10% | 全流程是否跑通，操作是否流畅 |

## 总体赛事要求

- 必须交付可运行的实物，不接受纯 PPT 演示
- 必须提交 AI 工具使用日志（用了哪些工具、解决了哪些问题，无日志不得答辩）
- 技术路线不限：Cursor / Claude Code / GitHub Copilot 任意
- 答卷整理至一份压缩包，命名 "姓名-题目"
